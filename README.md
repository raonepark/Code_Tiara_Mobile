# Code Tiara Mobile

React Native (Expo) version of Code Tiara.

---

## 🎨 Design Requirements (중요 디자인 지침)

### 1. 로그인 화면 디자인 (Login Screen Design)
- **모노크롬 화이트 테마 고정**: 로그인/회원가입/비밀번호 재설정 카드의 디자인은 PC 버전의 오리지널 디자인을 그대로 유지합니다.
  - **배경색**: 항상 밝은 그레이 톤 (`#F9FAFB` ~ `#F3F4F6` 계열)
  - **카드 배경**: 항상 흰색 (`#FFFFFF`)
  - **입력 폼 배경**: 연한 회색 (`#F2F2F2`) 및 어두운 글자색
  - **기본 버튼**: 항상 검은색 (`#000000`) 배경에 흰색 (`#FFFFFF`) 글씨
- **상단 토글 제거**: 로그인 화면에서 언어 선택 및 테마 선택 버튼은 배치하지 않습니다. 이 설정들은 본문 진입 후 전체 설정 창에서 관리됩니다.

### 2. 모바일 반응형 최적화 (Mobile-First Responsiveness)
- **기기별 렌더링 분기**:
  - **모바일 화면 (너비 < 450px)**: 데스크톱용 모달 테두리(Border), 외곽 섀도우(Shadow), 불필요한 배경 대비를 제거하여, 전체 화면이 부드럽고 자연스럽게 녹아드는 **모바일 네이티브 스타일(Full-screen Card)**로 렌더링되도록 구현해야 합니다.
  - **태블릿/데스크톱 화면 (너비 >= 450px)**: PC 버전처럼 중앙에 깔끔하게 떠 있는 플로팅 카드로 표현됩니다.

### 3. 사용자 추가 요청 사항 기록
- **게스트 모드 오류 수정**: 게스트 모드로 들어갔을 때 흰 공백 창으로 뜨는 문제를 수정합니다.
- **할일 목록(Todo List) 복구**: 임시 대시보드 대신 기존 PC 버전의 할일 목록 화면을 모바일에 맞게 이식하여 노출합니다.
- **제스처 최적화**: 마우스 오버나 더블 클릭 액션을 모바일 환경에 맞게 일반 터치나 길게 누르기(Long Press) 제스처로 변경합니다.

### 4. 컴포넌트 구현 및 안정성 지침
- **폰트 패밀리 제한**: React Native 환경에서 `fontFamily: 'System'`과 같이 일부 기기에서 정의되지 않은 폰트명을 직접 지정할 경우 렌더링 크래시가 발생할 수 있습니다. 폰트 스타일링 시 default로 두고 별도로 지정하지 않거나, Developer 테마인 경우에만 `'monospace'`로 렌더링되도록 분기 처리해야 합니다.

---

## 🌐 테마 및 다국어 준수 지침 (Theme & i18n Rules)
- **테마별 디자인 고려**: 모든 UI 변경 사항은 세 가지 테마(공주 테마, 엑셀 테마, 개발자/어두운 테마)의 고유한 색상 및 폰트 스타일을 모두 고려하여 테마별 분기 처리를 명확히 반영해야 합니다.
- **다국어 지원(i18n)**: 새로운 텍스트가 추가되거나 기존 텍스트가 변경될 때, 반드시 `ko.json` 및 `en.json`에 동시에 번역 키를 추가하여 반영해야 합니다. 하드코딩된 한국어/영어 텍스트 사용을 지양합니다.

---

## 🚀 실행 및 테스트 방법
현재 개발 서버가 켜져 있는 상태이며 아래 명령어로 브라우저 및 디바이스에서 실행할 수 있습니다.

```bash
# 개발 서버 시작 (웹 및 모바일 지원)
npx expo start

# 웹 버전으로 즉시 시작
npx expo start --web
```
